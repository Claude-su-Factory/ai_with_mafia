## ADDED Requirements

### Requirement: 방 생성
사용자는 방 이름, 최대 사람 인원수(1~6명), 공개/비공개 여부를 지정하여 방을 생성할 수 있어야 한다. 방을 생성한 사용자가 방장이 된다.

#### Scenario: 공개 방 생성
- **WHEN** 방 이름, max_humans(1~6), visibility=public으로 방 생성 요청을 보내면
- **THEN** 고유 ID를 가진 공개 방이 생성되고 방 정보가 반환되어야 한다

#### Scenario: 비공개 방 생성
- **WHEN** visibility=private으로 방 생성 요청을 보내면
- **THEN** 6자리 랜덤 영숫자 참가 코드가 생성되어 방 정보에 포함되어야 한다

#### Scenario: 잘못된 max_humans 값
- **WHEN** max_humans가 1 미만이거나 6 초과이면
- **THEN** 422 오류를 반환해야 한다

---

### Requirement: 방 참가
사용자는 공개 방 목록 또는 비공개 코드로 방에 참가할 수 있어야 한다.

#### Scenario: 공개 방 목록 조회
- **WHEN** 방 목록 API를 요청하면
- **THEN** 공개 방만 목록에 포함되어야 한다

#### Scenario: 비공개 방 코드 참가
- **WHEN** 6자리 참가 코드로 방 참가 요청을 보내면
- **THEN** 해당 비공개 방에 참가할 수 있어야 한다

#### Scenario: 잘못된 비공개 코드
- **WHEN** 존재하지 않는 코드로 참가 요청을 보내면
- **THEN** 404 오류를 반환해야 한다

#### Scenario: 정원 초과 참가
- **WHEN** max_humans 인원이 이미 차 있는 방에 참가 요청을 보내면
- **THEN** 409 오류를 반환해야 한다

---

### Requirement: 방장 관리
방장은 게임 시작 권한을 가진다. 방장 이탈 시 권한이 자동 이전되어야 한다.

#### Scenario: 방장 게임 시작
- **WHEN** 방장이 게임 시작 요청을 보내면
- **THEN** 게임이 시작되어야 한다 (참가 인원이 1명이어도 시작 가능)

#### Scenario: 비방장 게임 시작 시도
- **WHEN** 방장이 아닌 플레이어가 게임 시작 요청을 보내면
- **THEN** 403 오류를 반환해야 한다

#### Scenario: 방장 이탈 시 권한 이전
- **WHEN** 방장이 방에서 이탈하면
- **THEN** 입장 순서 기준 다음 사람 플레이어에게 방장 권한이 자동 이전되어야 한다

---

### Requirement: 플레이어 이탈 처리
게임 중 플레이어가 이탈하면 AI로 자동 대체되며, 모든 사람 플레이어가 이탈하면 게임이 종료된다.

#### Scenario: 게임 중 플레이어 이탈
- **WHEN** 게임 진행 중 사람 플레이어의 WebSocket 연결이 끊어지면
- **THEN** 해당 플레이어가 AI로 대체되고 전체 플레이어에게 대체 알림이 전송되어야 한다

#### Scenario: 모든 사람 플레이어 이탈
- **WHEN** 게임 중 마지막 사람 플레이어가 이탈하면
- **THEN** 게임이 즉시 종료되어야 한다

---

### Requirement: 빈 방 자동 삭제
방에 플레이어가 아무도 없으면 해당 방이 자동으로 삭제되어야 한다.

#### Scenario: 마지막 플레이어 이탈
- **WHEN** 대기 중인 방에서 마지막 플레이어가 이탈하면
- **THEN** 해당 방이 삭제되어야 한다

---

### Requirement: 게임 재시작
게임 종료 후 방장은 같은 방에서 게임을 다시 시작할 수 있어야 한다.

#### Scenario: 재시작
- **WHEN** 게임 종료 후 방장이 재시작 요청을 보내면
- **THEN** 페르소나와 역할이 새로 랜덤 배정되어 새 게임이 시작되어야 한다

---

### Requirement: WebSocket 실시간 통신
방에 참가한 플레이어는 WebSocket으로 게임 이벤트를 실시간으로 주고받을 수 있어야 한다.

#### Scenario: WebSocket 연결
- **WHEN** 유효한 플레이어가 WebSocket 엔드포인트에 연결하면
- **THEN** 연결이 수립되고 현재 방 상태가 전송되어야 한다

#### Scenario: 메시지 브로드캐스트
- **WHEN** 한 플레이어가 채팅 메시지를 전송하면
- **THEN** 같은 방의 모든 연결된 클라이언트에게 해당 메시지가 전달되어야 한다

---

### Requirement: Entity와 DTO 분리
도메인 Entity는 직접 직렬화하지 않고, DTO를 통해서만 외부에 노출되어야 한다.

#### Scenario: 방 정보 응답
- **WHEN** 방 정보 API를 호출하면
- **THEN** sync.Mutex나 channel 같은 내부 필드 없이 직렬화 가능한 DTO 형태로 응답해야 한다
